# Old UI Feature Bridge Migration

UIEditor_new must keep the old UIEditor shell and migrate every UI-affecting old feature to the Unity Editor Bridge / temporary Prefab flow. The web store is only the interaction mirror; Unity working Prefab state is authoritative.

## Status Legend

- `done`: routed to Bridge and included in build.
- `partial`: routed for common cases, still needs sample verification or deeper Unity component coverage.
- `pending`: old UI entry exists but still needs Bridge implementation.
- `hidden`: old JSON/local workflow is intentionally removed from the main user flow.
- `conflict`: old UI entry depends on the old JSON/WebGL-authoritative flow and should not be moved as-is.

## Uncertain / Needs User Or Sample Confirmation

| Area | Why It Is Uncertain | Proposed Handling |
| --- | --- | --- |
| Rotate / scale / transform gizmo behavior on complex nested samples | Basic browser gesture behavior is verified through Bridge, but complex pivots/anchors and nested layout groups can still differ from simple generated-node samples. | Keep the left toolbar row `done`; verify complex user-selected Prefab samples before removing this uncertainty. |
| Image mirror type | There is no confirmed Unity Bridge field contract for the old mirror behavior and no protected-diff whitelist proving it should be saved. | Hide the mirror editor from the Bridge main property panel until the Unity-side representation is confirmed; keep it in uncertain list instead of pretending it is saved. |
| Text gradient effect | There is no confirmed Unity component/field mapping in the Bridge visual patch whitelist. | Hide the gradient editor from the Bridge main property panel until the Unity-side component contract is defined. |
| Sprite border / sliced border editing | The picker can infer `sliceBorder` metadata, but saving sprite border values may require asset import settings rather than a Prefab visual field. | Keep Image.type / sprite replacement Bridge-backed; hide sliced border numeric editing until Unity-side semantics are confirmed. |
| Button Image component presence | Toggling whether a Button has an Image component changes component structure, not just a visual field. | Keep it read-only in the property panel; use Image enabled/color/sprite fields when an Image component already exists. |
| Generic background style | Bridge-backed color fields are component-specific (`Image.color`, `Text.color`, etc.), while the generic Web style background has no single Unity owner. | Hide the generic background editor from the Bridge main property panel; prefer Image.color on nodes that actually have an Image component. |

## Conflict / Do Not Move As-Is

| Old Behavior | Conflict With Current Flow |
| --- | --- |
| WebGL runtime as authoritative render target | New plan makes Unity Editor Bridge / temporary Prefab the authoritative visual state; WebGL runtime can only be diagnostic. |
| JSON full-tree sync as the main edit state | Web only mirrors artboard/session/selection/viewport and sends commands; Unity working Prefab is the state owner. |
| Thumbnail generation from current temporary edit canvas | User requirement says preview thumbnails update only after saving modified Prefab state, not from live temp Prefab edits. |
| Local save/load as the primary persistence path | Main persistence is `save-artboard` into a Unity Prefab path; old full-tree local autosave/load is hidden and disabled in startup recovery so it cannot replace Unity-owned Prefab state. |
